# 迭代二：单列车增强 + 扩展视图 — 需求文档

> **当前状态：🔄 开发中** — 迭代一（MVP）已全部完成，迭代二正在开发中。
>
> **目标：** 在迭代一（MVP 单列车从 A 站到 B 站闭环）的基础上，**增强后端仿真精度**（能耗计算、供电网压变化、轨道道岔与占用检测），**扩展前端视图**（供电、信号、轨道三个专业化视图），并补齐**数据导出与参数管理**等辅助功能。
>
> **总原则：** 迭代一验证了全链路可行性，迭代二将各子系统从"能用"提升到"基本可用"——供电不再固定网压、轨道不再只有直线、车辆不再忽略能耗、前端不再只有两个视图。

#### 2.2.3 欧姆压降计算逻辑

```
每步计算：

1. 确定列车当前所在供电区间（最近的两个变电所之间）
2. 计算列车到最近变电所的距离 d (km)
3. 计算接触网+钢轨回路总电阻 R = (r_contact + r_rail) × d
4. 计算列车电流 I = P_demand / V_nominal (P_demand 来自车辆系统)
5. 计算网压跌落 ΔV = I × R
6. 受电弓端电压 V_panto = V_nominal - ΔV

约束：
- V_panto 不得低于 1000V（低于此值触发欠压告警）
- 变电所输出功率不得超过额定容量
- 单变电所供电，不模拟双边供电/越区供电
```

#### 2.2.4 再生制动统计

```
每步（列车处于制动工况时）：

1. 获取当前制动力 F_brake (N) 和速度 v (m/s)
2. 计算制动功率 P_brake = F_brake × v (W)
3. 按再生制动效率 η=0.3 折算再生电量：
   E_regen += P_brake × η × Δt (J)
4. 牵引能耗累计（与 VHC-09 联动）：
   E_traction += P_traction × Δt (J) （牵引工况时）
```

**迭代二不实现：** 多车就近吸收再生能量、阶梯网压保护、断路器跳闸逻辑、越区供电。

### 2.3 车辆系统增强（Vehicle System Enhancement）

> **迭代一回溯：** VHC-01~VHC-10 实现了牛顿动力学解算、牵引/制动/阻力、限速约束、对标停车，但能耗计算（VHC-09）仅预留字段未实现。
>
> **迭代二新增：** 能耗累计（牵引能耗 + 再生制动电量）。

#### 2.3.1 功能清单

| 编号 | 功能 | 说明 | 优先级 | 迭代 | 状态 |
|------|------|------|--------|------|------|
| VHC-09 | 能耗计算 | 累计牵引能耗、再生制动电量（迭代一仅为预留接口，迭代二实现具体逻辑） | P0 | 二 | |

#### 2.3.2 能耗计算逻辑

```
牵引工况（traction_level > 0）：
  P_traction = F_traction × v (W)
  E_traction += P_traction × Δt (J)

制动工况（brake_level > 0）：
  P_brake = F_brake × v (W)
  E_regen += P_brake × regeneration_efficiency × Δt (J)

惰行/停车工况：
  不累计

单位转换：
  1 kWh = 3.6 × 10^6 J
```

> **迭代一回溯：** 迭代一实现了 SIG-01~SIG-03（三段式运行模式、站停时间控制、制动触发点自动计算），采用开环位置触发方式，无安全防护、无自动驾驶闭环调节。

#### 2.4.2 ATP 安全包络（SIG-04）

```
功能：
  1. 列车运行时，实时计算当前安全包络终点（当前位置 + 固定安全距离）
  2. 若包络终点触及前方障碍（站台终点、前车尾端等），触发紧急制动
  3. 安全距离默认 300m，可配置

输入：
  - 列车当前位置（来自车辆系统）
  - 前方障碍物位置（前车尾端、站台终点、线路终点）

输出：
  - 紧急制动指令（当安全包络侵入障碍物时）
  - 紧急制动状态（正常/触发中/已触发）

简化约定：
  - 单列车时安全包络终点恒为前方站台位置（或线路终点），不触发紧急制动
  - 多车时安全包络终点取前车尾端位置，侵入时触发紧急制动
  - 不计算动态速度曲线（MA曲线留待迭代三精细化）
```

#### 2.4.3 ATO 自动驾驶（SIG-05）

```
功能：
  1. 以目标速度（当前区段限速 × 速度系数）为跟踪目标
  2. PID 控制器根据速度偏差 e(t) = v_target - v_actual 调节牵引/制动级位
  3. 输出级位范围：[-1, 1]（负值表示制动，正值表示牵引）

PID 参数：
  - Kp: 比例系数（默认 0.5，可配置）
  - Ki: 积分系数（默认 0.05，可配置）
  - Kd: 微分系数（默认 0.1，可配置）
  - 输出限幅：[-1, 1]

输入：
  - 当前速度 v_actual（来自车辆系统）
  - 目标速度 v_target（来自 ATS 或线路限速）

输出：
  - 牵引级位 [0, 1]（正值）
  - 制动级位 [0, 1]（正值，PID 输出为负时取绝对值）

与迭代一三段式的关系：
  - 迭代二引入 ATO 后，三段式作为默认后备模式
  - 可通过配置切换信号模式：signal_mode = "ato" / "three_stage"
  - ATO 模式下，三段式逻辑不执行
```

#### 2.4.4 ATS 运行图调整（SIG-06）

```
功能：
  1. 列车到站对标停车后，记录实际到站时间
  2. 对比计划到站时间，计算晚点时间 delay = t_actual - t_plan
  3. 根据晚点时间调整本站停时间：
     - 晚点（delay > 0）：站停时间 = 原站停时间 - min(delay, 最大缩短量)
     - 早点（delay < 0）：站停时间 = 原站停时间 + min(-delay, 最大延长量)
  4. 最大调整量默认 ±10s，可配置

输入：
  - 计划到站时间（来自时刻表）
  - 实际到站时间（来自车辆系统对标信号）
  - 原站停时间（来自时刻表）

输出：
  - 调整后的站停时间
  - 时刻表偏差（晚点/早点秒数，供前端展示）

简化约定：
  - 仅调整站停时间，不调整区间运行等级
  - 晚点超出最大调整量时，剩余晚点累积到后续车站
```

#### 2.4.5 多车追踪间隔（SIG-07）

```
功能：
  1. 后车追踪前车时，保持固定安全距离（默认 500m）
  2. 若后车与前车距离 ≤ 安全距离，后车触发 ATP 紧急制动
  3. 前车离站后，后车进站前检查区间是否空闲

输入：
  - 前车位置（来自车辆系统或仿真编排器）
  - 后车位置（来自车辆系统）

输出：
  - 紧急制动指令（当追踪间隔不足时）
  - 发车延迟指令（前车未离站时后车禁止发车）

简化约定：
  - 固定安全距离，不基于前车速度动态计算
  - 单列车时不触发追踪逻辑
```
