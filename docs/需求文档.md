## 一、项目概述

### 1.1 产品定位

一套城市轨道交通运行仿真系统，用于模拟地铁线路中**列车运行、供电潮流、信号控制和轨道基础设施**的协同工作过程。系统以仿真引擎为核心，配合前端可视化界面，实现对列车运行状态、子系统运行参数的实时监控与分析。

### 1.2 目标用户

| 用户角色 | 核心诉求 |
|---------|---------|
| 轨道交通系统工程师 | 验证列车运行控制逻辑、供电方案、信号系统设计 |
| 运营规划人员 | 评估运行图可行性、测试晚点恢复策略、优化发车间隔 |
| 教学/科研人员 | 观察各子系统间的耦合关系、进行仿真实验和数据采集 |

## 二、子系统功能需求

### 2.1 轨道系统 (Track System)

#### 2.1.1 功能描述

将地铁线路数字化为一维公里标路径，为其他子系统提供空间位置查询服务，包括坡度、曲率、限速、车站位置、隧道区段等信息。

#### 2.1.2 功能清单

| 编号 | 功能 | 说明 | 优先级 | 迭代 | 状态 |
|------|------|------|--------|------|------|
| TRK-01 | 一维线性路径建模 | 以车站和区间分段定义全线路径，每段标注坡度、曲率、限速 | P0 | 一 | |
| TRK-02 | 位置查询服务 | 给定公里标，返回坡度、曲率、限速、隧道标识 | P0 | 一 | |
| TRK-03 | 车站位置查询 | 给定车站 ID，返回站台中心公里标 | P0 | 一 | |
| TRK-04 | 站台范围判断 | 给定公里标，判断是否在站台范围内（±15m） | P0 | 一 | |
| TRK-05 | 多线路支持 | 支持上行/下行双方向路径 | P1 | 二 | |
| TRK-06 | 道岔建模 | 道岔几何参数、定位/反位状态、侧向限速 | P1 | 二 | |
| TRK-07 | 轨道电路/计轴器 | 区段占用/出清检测 | P1 | 二 | |
| TRK-09 | 外部数据导入 | 从 CSV 导入线路设计数据（暂不支持 DXF） | P2 | 三 | |

#### 2.1.3 数据接口

```python
# 输入
class TrackInput:
    line_design: LineDesign     # 线路设计参数（坡度、曲率、车站、限速等）
    switch_commands: list       # 道岔控制指令（来自信号系统）

# 输出
class TrackOutput:
    resistance_params: ResistanceParams   # 坡度阻力、弯道阻力、隧道阻力系数
    switch_status: SwitchStatus           # 道岔状态（密贴/锁闭/转换中）
    section_occupancy: dict               # 轨道区段占用状态
    station_platform: StationPlatform     # 站台位置信息
```

### 2.2 车辆系统 (Vehicle System) — 核心仿真模块

#### 2.2.1 功能描述

模拟列车在轨道上的动力学行为，包括牵引、惰行、制动三种工况，计算加速度、速度、位置随时间的变化，并输出能量需求。

#### 2.2.2 功能清单

| 编号 | 功能 | 说明 | 优先级 | 迭代 | 状态 |
|------|------|------|--------|------|------|
| VHC-01 | 牛顿第二定律动力学解算 | F=ma，计算合力→加速度→速度→位置 | P0 | 一 | |
| VHC-02 | 牵引特性曲线 | 分段线性模型，速度-牵引力映射 | P0 | 一 | |
| VHC-03 | 基本阻力计算 | Davis 公式：R = A + Bv + Cv² | P0 | 一 | |
| VHC-04 | 坡度附加阻力 | 从轨道系统获取坡度，计算 Rg = mg·sinθ | P0 | 一 | |
| VHC-05 | 弯道附加阻力 | 从轨道系统获取曲率，计算 Rc | P0 | 一 | |
| VHC-06 | 隧道空气阻力 | 隧道区段空气阻力系数加成 | P0 | 一 | |
| VHC-07 | 限速约束 | 速度不得超过所在区段限速 | P0 | 一 | |
| VHC-08 | 站台对标停车 | 在站台中心 ±1m 内停车 | P0 | 一 | |
| VHC-09 | 能耗计算 | 累计牵引能耗、再生制动电量 | P1 | 二 | |
| VHC-13 | 乘客载荷模型 | 随机上下客模拟质量变化，每次站停后按随机比例更新载客量 | P2 | 三 | |

#### 2.2.3 数据接口

```python
# 输入
class VehicleInput:
    vehicle_params: VehicleParams       # 车辆固有参数（质量、牵引特性等）
    control_commands: ControlCommands   # 牵引/制动级位（来自信号系统）
    track_params: TrackResistanceParams # 坡度、曲率、限速（来自轨道系统）
    pantograph_voltage: float           # 受电弓端电压（来自供电系统）

# 输出
class VehicleOutput:
    position: float                     # 实时公里标（m）
    speed: float                        # 瞬时速度（km/h）
    acceleration: float                 # 瞬时加速度（m/s²）
    mode: str                           # 当前工况：牵引/惰行/制动
    power_demand: float                 # 受电弓功率请求（kW，传给供电系统）
    axle_speed: float                   # 轴端脉冲速度（传给信号系统）
    door_status: DoorStatus             # 车门状态（传给信号系统）
    fault_alarm: FaultAlarm             # 故障告警（传给信号系统）
```

### 2.3 供电系统 (Power Supply System)

#### 2.3.1 功能描述

模拟全线供电网络（变电所→接触网/第三轨→列车→钢轨回流）的电气行为，包括潮流计算、再生制动能量吸收、网压波动。

#### 2.3.2 功能清单

| 编号 | 功能 | 说明 | 优先级 | 迭代 | 状态 |
|------|------|------|--------|------|------|
| PWR-01 | 固定网压输出 | 受电弓端电压恒为 DC 1500V | P0 | 一 | |
| PWR-02 | 简单欧姆压降计算 | 根据列车位置和电流估算网压跌落 | P1 | 二 | |
| PWR-03 | 变电所模型 | 整流外特性曲线、输出电压/电流 | P1 | 二 | |
| PWR-04 | 再生制动能量吸收 | 单车独立统计，按固定效率（默认30%）计算再生电量，不模拟多车就近吸收 | P1 | 二 | |
| PWR-05 | 双向能耗流动 | 牵引工况（耗电）与再生制动（馈电）双向模拟 | P2 | 三 | |
| PWR-06 | 阶梯网压保护 | 断路器、隔离开关跳闸及越区供电逻辑（查表简化版，不模拟保护时序配合） | P2 | 三 | |

#### 2.3.3 数据接口

```python
# 输入
class PowerInput:
    substation_params: list             # 变电所参数（外特性曲线、位置）
    network_params: NetworkParams       # 接触网/钢轨阻抗参数
    train_loads: list[TrainLoad]        # 各列车位置 + 功率请求（来自车辆系统）

# 输出
class PowerOutput:
    substation_states: list             # 各变电所电流/有功/无功/能耗
    catenary_voltage_profile: list      # 全线接触网电压分布
    train_pantograph_voltage: dict      # 各列车受电弓端电压（→车辆系统）
    regeneration_energy: float          # 再生制动能量利用率
```

### 2.4 信号系统 (Signaling System)

#### 2.4.1 功能描述

实现列车运行控制的核心逻辑，包括 ATP 安全防护、ATO 自动驾驶、ATS 运行图调度和联锁（CI）逻辑。

#### 2.4.2 功能清单

| 编号 | 功能 | 说明 | 优先级 | 迭代 | 状态 |
|------|------|------|--------|------|------|
| SIG-01 | 三段式运行模式 | 牵引→惰行→制动，按位置触发切换 | P0 | 一 | |
| SIG-02 | 站停时间控制 | 到站停车后等待设定时间再发车 | P0 | 一 | |
| SIG-03 | 制动触发点自动计算 | 根据当前速度+减速度能力反推制动点 | P0 | 一 | |
| SIG-04 | ATP 安全包络 | 固定安全距离模型（默认300m触发紧急制动），不计算动态MA速度曲线 | P1 | 二 | |
| SIG-05 | ATO 自动驾驶 | PID 控制替代 MPC，自动调节牵引/制动级位 | P1 | 二 | |
| SIG-06 | ATS 运行图调整 | 晚点检测后仅调整站停时间（延长/缩短），不做区间运行等级调整 | P1 | 二 | |
| SIG-07 | 多车追踪间隔 | 固定安全距离追踪（默认500m），不基于前车位置动态计算最小间隔 | P1 | 二 | |
| SIG-08 | 联锁逻辑（CI） | 进路-道岔-信号机互锁检查（简化版：仅做静态进路表查表，不模拟实时互锁时序） | P2 | 三 | |
| SIG-09 | 通信时延模拟 | 无线系统延时抖动和丢包率模拟 | P2 | 三 | |
| SIG-10 | 紧急制动触发 | ATP 超速防护、车门未锁闭防护 | P2 | 三 | |

#### 2.4.3 数据接口

```python
# 输入
class SignalInput:
    timetable: Timetable                 # 电子运行图、站停时间、交路表
    train_feedback: list[TrainFeedback]  # 各列车位置/速度/车门状态（来自车辆系统）
    track_feedback: TrackFeedback        # 计轴占用/道岔状态（来自轨道系统）
    communication_status: CommStatus     # 无线通信延时/丢包率

# 输出
class SignalOutput:
    control_commands: list[TrainCommand] # 牵引/制动级位（→车辆系统）
    emergency_brake: list[EmergencyBrake]# 紧急制动命令（→车辆系统）
    door_permit: DoorPermit              # 开/闭车门允许信号（→车辆系统）
    route_requests: list[RouteRequest]   # 进路请求/道岔控制（→轨道系统）
    ats_display: ATSDisplay              # 全线调度画面数据
    train_interval: float                # 当前发车间隔（s）
```

## 五、子系统间数据流

### 5.1 完整数据流图

```
┌──────────────┐    牵引/制动级位指令    ┌──────────────┐
│              │ ─────────────────────► │              │
│  信号系统    │                         │  车辆系统     │
│              │ ◄─────────────────────  │              │
│  ATP/ATO/ATS │  实时位置/速度/车门状态  │  动力学解算   │
│  联锁        │                         │              │
└──────┬───────┘                         └──────┬───────┘
       │                                        │
       │ 道岔控制指令                            │ 功率请求
       ▼                                        ▼
┌──────────────┐                         ┌──────────────┐
│  轨道系统     │                        │  供电系统     │
│              │ ◄────────────────────── │              │
│  拓扑/阻力    │   受电弓端电压          │  潮流计算     │
│  道岔/计轴    │                        │  变电所模型   │
└──────────────┘                         └──────────────┘
       │                                       │
       └──────────┬────────────────────────────┘
                  │
                  ▼
          ┌──────────────┐
          │  数据记录器   │
          │  CSV / 实时流 │
          └──────┬───────┘
                 │
                 ▼
          ┌──────────────┐
          │  前端可视化   │
          │  WebSocket   │
          └──────────────┘
```

### 5.2 每个仿真步的数据流顺序

```
Step N:
  1. 时钟推进：t = t + Δt
  2. 信号系统：根据当前列车状态计算新的控制指令
  3. 轨道系统：查询各列车当前位置的线路参数
  4. 车辆系统：结合控制指令 + 线路参数 + 网压，解算动力学
  5. 供电系统：根据各列车功率需求，计算网压跌落
  6. 数据记录：将当前步状态写入缓冲区
  7. 状态推送：通过 WebSocket 发送到前端
```
