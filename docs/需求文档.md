## 一、项目概述

### 1.1 产品定位

一套城市轨道交通运行仿真系统，用于模拟地铁线路中**列车运行、供电潮流、信号控制和轨道基础设施**的协同工作过程。系统以仿真引擎为核心，配合前端可视化界面，实现对列车运行状态、子系统运行参数的实时监控与分析。

### 1.2 目标用户

| 用户角色 | 核心诉求 |
|---------|---------|
| 轨道交通系统工程师 | 验证列车运行控制逻辑、供电方案、信号系统设计 |
| 运营规划人员 | 评估运行图可行性、测试晚点恢复策略、优化发车间隔 |
| 教学/科研人员 | 观察各子系统间的耦合关系、进行仿真实验和数据采集 |

## 子系统概览

下表从**输入、输出、仿真优化目标**三个维度，对各子系统进行简要说明，帮助读者快速建立系统整体认知。

| 子系统 | 核心职责 | 主要输入 | 主要输出 | 仿真优化目标 |
|--------|----------|----------|----------|-------------|
| **轨道系统**<br>(Track System) | 将地铁线路数字化为一维公里标路径，为其他子系统提供空间位置查询服务；管理站台屏蔽门（PSD）及站台紧急停车按钮状态 | 线路设计参数（坡度、曲率、限速、车站位置）、道岔控制指令、PSD 开/闭命令 | 坡度/弯道/隧道阻力系数、道岔状态、区段占用状态、站台位置信息、PSD 状态、EPB 状态 | 准确提供全线任意公里标的几何与限制参数查询，正确反映道岔/区段占用逻辑，正确模拟 PSD 与 EPB 状态机 |
| **车辆系统**<br>(Vehicle System) | 模拟列车在轨道上的动力学行为，计算牵引/惰行/制动工况下的运动状态与能量需求；管理列车车门时序，与 PSD 联动 | 车辆固有参数（质量、牵引特性曲线）、牵引/制动级位、轨道阻力参数、受电弓端电压、开/闭车门命令、发车许可 | 实时位置（公里标）、速度、加速度、当前工况、受电弓功率请求、轴端脉冲速度、车门状态、车门故障详情、PSD 同步状态 | 精确的牛顿动力学解算（F=ma），站台对标停车误差 ≤ ±1m，合理反映轮轨粘着与附加阻力，正确执行车门-PSD 偏序门控逻辑，在保持乘客舒适度的情况下提升效率 |
| **供电系统**<br>(Power Supply System) | 模拟变电所→接触网→列车→钢轨回流的电气行为，计算潮流分布与网压波动 | 变电所参数（外特性曲线、位置）、接触网/钢轨阻抗参数、各列车位置 + 功率请求 | 变电所电流/有功/无功、全线接触网电压分布、各列车受电弓端电压、再生制动能量利用率 | 稳定输出受电弓端电压（DC 1500V），合理模拟欧姆压降，统计再生制动能量吸收效率 |
| **信号系统**<br>(Signaling System) | 实现列车运行控制核心逻辑，包括 ATP 安全防护、ATO 自动驾驶、ATS 运行图调度、联锁、时刻表管理、发车条件联控及站台紧急停车处理 | 列车位置/速度/车门状态/轴端脉冲速度/车门故障详情、轨道区段占用状态、道岔状态、PSD 状态、EPB 状态、时刻表数据 | 牵引/制动级位、目标速度、紧急制动指令、站停/发车指令、进路排列指令、车门/PSD 联动命令、发车许可、时刻表偏差数据 | 保证安全包络下不发生追尾/超速，ATO 自动调节级位实现平稳运行，ATS 按运行图调整站停时间，发车联控确保车门→PSD→信号全部就绪后方可发车 |

## 二、子系统功能需求

### 2.1 轨道系统 (Track System)

#### 2.1.1 功能描述

将地铁线路数字化为一维公里标路径，为其他子系统提供空间位置查询服务，包括坡度、曲率、限速、车站位置、隧道区段等信息。

#### 2.1.2 功能清单

| 编号 | 功能 | 说明 | 优先级 | 迭代 | 状态 |
|------|------|------|--------|------|------|
| TRK-01 | 一维线性路径建模 | 以车站和区间分段定义全线路径，每段标注坡度、曲率、限速 | P0 | 一 | 已完成 |
| TRK-02 | 位置查询服务 | 给定公里标，返回坡度、曲率、限速、隧道标识 | P0 | 一 | 已完成 |
| TRK-03 | 车站位置查询 | 给定车站 ID，返回站台中心公里标 | P0 | 一 | 已完成 |
| TRK-04 | 站台范围判断 | 给定公里标，判断是否在站台范围内（±15m） | P0 | 一 | 已完成 |
| TRK-05 | 多线路支持 | 支持上行/下行双方向路径 | P1 | 二 | 开发中 |
| TRK-06 | 道岔建模 | 道岔几何参数、定位/反位状态、侧向限速 | P1 | 二 | 开发中 |
| TRK-07 | 轨道电路/计轴器 | 区段占用/出清检测 | P1 | 二 | 开发中 |
| TRK-09 | 外部数据导入 | 从 CSV 导入线路设计数据（暂不支持 DXF） | P2 | 三 | |
| TRK-10 | 站台屏蔽门（PSD）建模 | 每站站台侧布置 N 对屏蔽门，含门状态（开/关/故障/旁路） | P1 | 三 | |
| TRK-11 | 站台停车区段建模 | 定义站台对标停车区段（±15m），用于 PSD 与车门对齐判断 | P1 | 三 | |

#### 2.1.3 数据接口

```python
# 输入
class TrackInput:
    line_design: LineDesign     # 线路设计参数（坡度、曲率、车站、限速等）
    switch_commands: list       # 道岔控制指令（来自信号系统）

# 输出
class TrackOutput:
    resistance_params: ResistanceParams   # 坡度阻力、弯道阻力、隧道阻力系数
    switch_status: SwitchStatus           # 道岔状态（密贴/锁闭/转换中）
    section_occupancy: dict               # 轨道区段占用状态
    station_platform: StationPlatform     # 站台位置信息
    psd_status: dict[str, PSDStatus]      # 各站台屏蔽门状态（→信号系统/前端）
    epb_status: dict[str, EPBStatus]      # 各站台紧急停车按钮状态（→信号系统/前端）
```

### 2.2 车辆系统 (Vehicle System) — 核心仿真模块

#### 2.2.1 功能描述

模拟列车在轨道上的动力学行为，包括牵引、惰行、制动三种工况，计算加速度、速度、位置随时间的变化，并输出能量需求。

#### 2.2.2 功能清单

| 编号 | 功能 | 说明 | 优先级 | 迭代 | 状态 |
|------|------|------|--------|------|------|
| VHC-01 | 牛顿第二定律动力学解算 | F=ma，计算合力→加速度→速度→位置 | P0 | 一 | 已完成 |
| VHC-02 | 牵引特性曲线 | 分段线性模型，速度-牵引力映射 | P0 | 一 | 已完成 |
| VHC-03 | 基本阻力计算 | Davis 公式：R = A + Bv + Cv² | P0 | 一 | 已完成 |
| VHC-04 | 坡度附加阻力 | 从轨道系统获取坡度，计算 Rg = mg·sinθ | P0 | 一 | 已完成 |
| VHC-05 | 弯道附加阻力 | 从轨道系统获取曲率，计算 Rc | P0 | 一 | 已完成 |
| VHC-06 | 隧道空气阻力 | 隧道区段空气阻力系数加成 | P0 | 一 | 已完成 |
| VHC-07 | 限速约束 | 速度不得超过所在区段限速 | P0 | 一 | 已完成 |
| VHC-08 | 站台对标停车 | 在站台中心 ±1m 内停车 | P0 | 一 | 已完成 |
| VHC-09 | 能耗计算 | 累计牵引能耗、再生制动电量 | P1 | 二 | 开发中 |
| VHC-10 | 乘客载荷模型 | 随机上下客模拟质量变化，每次站停后按随机比例更新载客量 | P2 | 三 | |
| VHC-11 | 列车车门控制 | 到站停车后执行开门/关门时序，车门状态（开/关/正在开关/故障/旁路）由信号系统 door_permit 触发 | P1 | 三 | |
| VHC-12 | 车门与 PSD 联动 | 收到信号系统 PSD_door_permit 后，车门与站台屏蔽门同步开关（偏序：PSD先开→车门开→关门→PSD后关） | P1 | 三 | |
| VHC-13 | 车门故障模式 | 模拟单扇或多扇车门无法开启/关闭（故障率可配置），关门故障触发联控锁闭 | P1 | 三 | |

#### 2.2.3 数据接口

```python
# 输入
class VehicleInput:
    vehicle_params: VehicleParams       # 车辆固有参数（质量、牵引特性等）
    control_commands: ControlCommands   # 牵引/制动级位（来自信号系统）
    track_params: TrackResistanceParams # 坡度、曲率、限速（来自轨道系统）
    pantograph_voltage: float           # 受电弓端电压（来自供电系统）
    door_command: DoorCommand           # 开门/关门命令（来自信号系统）
    departure_permit: bool              # 发车许可（来自信号系统）

# 输出
class VehicleOutput:
    position: float                     # 实时公里标（m）
    speed: float                        # 瞬时速度（km/h）
    acceleration: float                 # 瞬时加速度（m/s²）
    mode: str                           # 当前工况：牵引/惰行/制动
    power_demand: float                 # 受电弓功率请求（kW，传给供电系统）
    axle_speed: float                   # 轴端脉冲速度（传给信号系统）
    door_status: DoorStatus             # 车门状态（开/关/正在开关/故障/旁路，传给信号系统）
    door_fault_detail: DoorFaultDetail  # 车门故障详细信息（故障扇区编号、故障类型）
    psd_sync_status: PSD_SyncStatus     # 车门-PSD 同步状态（已对齐/等待开门/等待关门/失步）
    fault_alarm: FaultAlarm             # 故障告警（传给信号系统）
```

### 2.3 供电系统 (Power Supply System)

#### 2.3.1 功能描述

模拟全线供电网络（变电所→接触网/第三轨→列车→钢轨回流）的电气行为，包括潮流计算、再生制动能量吸收、网压波动。

#### 2.3.2 功能清单

| 编号 | 功能 | 说明 | 优先级 | 迭代 | 状态 |
|------|------|------|--------|------|------|
| PWR-01 | 固定网压输出 | 受电弓端电压恒为 DC 1500V | P0 | 一 | 已完成 |
| PWR-02 | 简单欧姆压降计算 | 根据列车位置和电流估算网压跌落 | P1 | 二 | 开发中 |
| PWR-03 | 变电所模型 | 整流外特性曲线、输出电压/电流 | P1 | 二 | 开发中 |
| PWR-04 | 再生制动能量吸收 | 单车独立统计，按固定效率（默认30%）计算再生电量，不模拟多车就近吸收 | P1 | 二 | 开发中 |
| PWR-05 | 双向能耗流动 | 牵引工况（耗电）与再生制动（馈电）双向模拟 | P2 | 三 | |
| PWR-06 | 阶梯网压保护 | 断路器、隔离开关跳闸及越区供电逻辑（查表简化版，不模拟保护时序配合） | P2 | 三 | |

#### 2.3.3 数据接口

```python
# 输入
class PowerInput:
    substation_params: list             # 变电所参数（外特性曲线、位置）
    network_params: NetworkParams       # 接触网/钢轨阻抗参数
    train_loads: list[TrainLoad]        # 各列车位置 + 功率请求（来自车辆系统）

# 输出
class PowerOutput:
    substation_states: list             # 各变电所电流/有功/无功/能耗
    catenary_voltage_profile: list      # 全线接触网电压分布
    train_pantograph_voltage: dict      # 各列车受电弓端电压（→车辆系统）
```

#### 2.4.4 车站联控逻辑详细说明

```
到站停车 → 开门联控（偏序） → 站停计时 → 关门联控（偏序） → 发车联控 → 发车

1. 开门联控（列车对标停稳后自动触发）：
   ① 信号系统发送 PSD 开门命令（→ 轨道系统）
   ② PSD 开启（模拟 2-3s 延迟）
   ③ 信号系统发送列车车门开门命令（→ 车辆系统）
   ④ 列车车门开启（模拟 2-3s 延迟）
   ⑤ 进入站停计时

2. 关门联控（站停时间到或 ATS 提前发车指令后触发）：
   ① 信号系统发送列车车门关门命令（→ 车辆系统）
   ② 列车车门关闭（模拟 2-3s 延迟）
   ③ 信号系统发送 PSD 关门命令（→ 轨道系统）
   ④ PSD 关闭（模拟 2-3s 延迟）

3. 发车条件联控（关门联控完成后触发）：
   ① 检查列车车门全部关闭 → 若不满足：输出告警，禁止发车
   ② 检查 PSD 全部关闭 → 若不满足：输出告警，禁止发车
   ③ 检查前方信号开放 → 若不满足：等待信号开放
   ④ 全部条件满足 → 信号系统释放发车许可（departure_permit）→ 车辆系统执行发车

4. 站台紧急停车按钮（EPB）：
   ① EPB 被触发 → 轨道系统感知并推送给信号系统
   ② 信号系统立即向该站台相关列车发出紧急制动命令（无论列车处于进站/停站/发车状态）
   ③ EPB 保持期间，发车条件联控持续阻塞
   ④ EPB 复位后，信号系统恢复正常，联控重新检查

5. 时刻表偏离计算：
   ① 列车对标停车后，记录实际到站时刻 t_arr_actual
   ② 对比计划到站时刻 t_arr_plan，计算晚点时间 delay = t_arr_actual - t_arr_plan（正=晚点，负=早点）
   ③ 列车实际发车时刻 t_dep_actual 对比计划发车时刻 t_dep_plan，同样计算偏差
   ④ 输出偏差数据供 ATS 调整站停时间及前端展示
```

### 5.2 每个仿真步的数据流顺序

```
Step N:
  1. 时钟推进：t = t + Δt
  2. 信号系统：根据当前列车状态计算新的控制指令
  3. 轨道系统：查询各列车当前位置的线路参数
  4. 车辆系统：结合控制指令 + 线路参数 + 网压，解算动力学
  5. 供电系统：根据各列车功率需求，计算网压跌落
  6. 数据记录：将当前步状态写入缓冲区
  7. 状态推送：通过 WebSocket 发送到前端
```

### 附录 A：术语表

| 术语 | 说明 |
|------|------|

| ATO | 列车自动驾驶系统，自动调节牵引/制动 |
| ATS | 列车自动监控系统，运行图管理和调度 |
| MA | 移动授权，ATP 计算的列车安全运行终点 |
| Davis 公式 | 列车基本阻力的经验公式 R = A + Bv + Cv² |
| 受电弓 | 列车从接触网取电的装置 |
| 再生制动 | 列车制动时电机反转发电，将动能转化为电能回馈电网 |
