# 迭代一：MVP 最小可行系统 — 需求文档

> **目标：** 在最短的时间内跑通「单列车从 A 站到 B 站」的完整仿真闭环，包含**后端仿真引擎 + 前端可视化界面**，验证全链路可行性。
>
> **总原则：** 后端只实现两个核心子系统（轨道 + 车辆），供电与信号全部用静态数据替代。前端实现核心视图和交互控制，不追求完美 UI，只追求逻辑闭环和可视化验证。

### 2.2 轨道系统（Track System）

> **设计原则：** 将真实的三维空间曲线简化为**一维公里标路径 + 分段参数表**。放弃毫米级精度，只保证区段级正确性。

#### 2.2.1 数据模型

```
Track:
  - stations: Station[]      # 车站列表（含站台中心里程）
  - segments: Segment[]      # 区间分段列表

Station:
  - id: string               # 车站 ID，如 "ST01"
  - name: string             # 车站名称，如 "北京站"
  - chainage: float          # 站台中心公里标（m）
  - dwell_time: float        # 站停时间（s）

Segment:
  - id: string               # 区段 ID
  - start_chainage: float    # 起点公里标（m）
  - end_chainage: float      # 终点公里标（m）
  - length: float            # 长度（m）= end - start
  - gradient: float          # 坡度（‰，上坡为正）
  - curvature: float         # 曲率半径（m，直线为 0 或无穷大）
  - speed_limit: float       # 最高限速（km/h）
  - tunnel: boolean          # 是否在隧道内
```

初始化时预设一条简单线路（不少于 3 站 2 区间），手动录入以上参数，不依赖外部文件读取。

#### 2.2.2 运行时查询接口

| 编号 | 功能 | 说明 | 优先级 | 迭代 | 状态 |
|------|------|------|--------|------|------|
| TRK-01 | 一维线性路径建模 | 以车站和区间分段定义全线路径，每段标注坡度、曲率、限速 | P0 | 一 | |
| TRK-02 | 位置查询服务 | 给定公里标，返回坡度、曲率、限速、隧道标识 | P0 | 一 | |
| TRK-03 | 车站位置查询 | 给定车站 ID，返回站台中心公里标 | P0 | 一 | |
| TRK-04 | 站台范围判断 | 给定公里标，判断是否在站台范围内（±15m） | P0 | 一 | |

**MVP 不实现：** 道岔任何逻辑、轨道电路/计轴器、三维曲线精确表达、竖曲线。

### 2.3 车辆系统（Vehicle System）

> **设计原则：** 实现最基本的单质点动力学模型。忽略轮轨粘着、空气制动、电气制动等复杂物理过程。

#### 2.3.1 数据模型

```
Vehicle:
  - id: string                          # 车辆 ID
  - mass: float                         # 当前总质量（kg, 含载客）
  - empty_mass: float                   # 空车质量（kg）
  - passenger_load: float               # 当前载客量（占满员比例 0.0~1.0）
  - max_traction_force: float           # 最大牵引力（N）
  - max_brake_force: float              # 最大制动力（N）
  - traction_curve: TractionCurve[]     # 牵引特性曲线（速度→牵引力百分比）

TractionCurvePoint:
  - speed: float                        # 速度（km/h）
  - force_percent: float                # 该速度下可用牵引力百分比（0~1.0）
```

#### 2.3.2 动力学解算

| 编号 | 功能 | 说明 | 优先级 | 迭代 | 状态 |
|------|------|------|--------|------|------|
| VHC-01 | 牛顿第二定律动力学解算 | F=ma，计算合力→加速度→速度→位置 | P0 | 一 | |
| VHC-02 | 牵引特性曲线 | 分段线性模型，速度-牵引力映射 | P0 | 一 | |
| VHC-03 | 基本阻力计算 | Davis 公式：R = A + Bv + Cv² | P0 | 一 | |
| VHC-04 | 坡度附加阻力 | 从轨道系统获取坡度，计算 Rg = mg·sinθ | P0 | 一 | |
| VHC-05 | 弯道附加阻力 | 从轨道系统获取曲率，计算 Rc | P0 | 一 | |
| VHC-06 | 隧道空气阻力 | 隧道区段空气阻力系数加成 | P0 | 一 | |
| VHC-07 | 限速约束 | 速度不得超过所在区段限速 | P0 | 一 | |
| VHC-08 | 站台对标停车 | 在站台中心 ±1m 内停车 | P0 | 一 | |

#### 2.3.3 牵引特性曲线（MVP 简化版）

采用分段线性模型（默认参数可配置）：

| 速度区间 (km/h) | 可用牵引力百分比 |
|----------------|----------------|
| 0 ~ 40         | 100%            |
| 40 ~ 80        | 100% → 50% 线性递减 |
| > 80           | 50%             |

#### 2.3.4 基本阻力参数（Davis 公式默认值，可配置）

| 参数 | 默认值 | 说明 |
|------|-------|------|
| A    | 0.01 × m × g | 滚动阻力系数 |
| B    | 0.0001 × m × g | 线性项系数 |
| C    | 0.5 × ρ × Cd × A_front | 空气阻力（ρ=1.2, Cd=0.5, A_front=10m²） |

| 编号 | 功能 | 说明 | 优先级 | 迭代 | 状态 |
|------|------|------|--------|------|------|
| VHC-09 | 能耗计算 | 累计牵引能耗、再生制动电量（预留接口，迭代一暂不实现具体逻辑） | P0 | 一 | |
| VHC-10 | 输出记录 | 每个仿真步记录：时间、位置、速度、加速度、工况、各阻力分量 | P0 | 一 | |

### 2.5 静态数据提供器 — 信号

| 编号 | 功能 | 说明 | 优先级 | 迭代 | 状态 |
|------|------|------|--------|------|------|
| SIG-01 | 三段式运行模式 | 牵引→惰行→制动，按位置触发切换 | P0 | 一 | |
| SIG-02 | 站停时间控制 | 到站停车后等待设定时间再发车 | P0 | 一 | |
| SIG-03 | 制动触发点自动计算 | 根据当前速度+减速度能力反推制动点 | P0 | 一 | |

#### 2.5.1 三段式运行模式详细逻辑

```
每个区间循环执行：

  1. 牵引阶段
     条件：v < v_target (v_target = 0.8 × speed_limit)
     动作：施加最大牵引力
     结束：v >= v_target → 进入惰行

  2. 惰行阶段
     条件：已牵引至目标速度，且距离前方站台 > 制动触发距离
     动作：F_traction = 0, F_brake = 0
     结束：当前位置 + 制动触发距离 >= 站台中心位置 → 进入制动

  3. 制动阶段
     条件：当前位置 + 制动触发距离 >= 站台中心位置
     动作：施加最大制动力
     结束：v <= 0 且 |x - station_center| < 1m → 停车
```

**MVP 不实现：** ATP 安全包络计算、ATO PID/MPC 控制、ATS 运行图调整与晚点恢复、手动驾驶模式、联锁逻辑、移动授权（MA）、紧急制动曲线。

## 四、验收场景（User Story）

### 场景 1：单列车一站直达（后端 + 前端联合验证）

```
给定：线路为 3 站 2 区间（A站→B站→C站），各区间参数已知
     列车空车质量 200t，AW2 载客（满员 60%）
     各区间限速 80km/h
     前端已连接后端 WebSocket
当：  用户点击"运行"按钮
则：  前端顶部栏时钟开始走动
     综合视图中：
       - 线路纵断面图显示三个车站和两个区间
       - 列车图标从 A 站出发沿线路移动
       - 速度-位置曲线实时绘制
       - 关键状态卡片更新数值
     车辆视图中：
       - 速度-时间曲线上升→平缓→下降
       - 加速度曲线显示正/负/零
       - 工况指示器：牵引→惰行→制动交替
     列车从 A 站出发
     → 牵引至 64km/h（80%限速）
     → 惰行一段距离
     → 制动减速至 B 站站台中心 ±1m 停车
     → 停车 30s（站停时间），时钟继续走动
     → 重复上述过程至 C 站
     → 仿真结束，状态栏显示"stopped"
```

### 场景 2：坡度影响验证（可视化对比）

```
给定：场景 1 的线路基础上，B站→C站区间加入 30‰ 上坡
当：  仿真启动
则：  车辆视图中速度-时间曲线在上坡段斜率明显变化
     车辆视图中加速度曲线在上坡段出现额外负向偏移
     与场景 1 的 B站→C站段运行时间应有可量化的差异
```

### 场景 3：仿真控制交互验证

```
给定：仿真正在运行
当：  用户点击"暂停"按钮
则：  仿真时钟停止，列车位置/速度冻结
     所有曲线和数值停止更新
当：  用户点击"继续"按钮
则：  仿真从暂停处继续推进
当：  用户点击"停止"按钮
则：  仿真结束，显示运行统计摘要
当：  用户选择 10× 倍率后重新运行
则：  仿真推进速度明显加快（10 分钟运营 ≈ 1 分钟完成）
```

### 场景 4：参数编辑验证

```
给定：仿真处于空闲或暂停状态
当：  用户在参数面板中将车辆质量从 200t 改为 220t
     然后启动仿真
则：  加速度曲线明显低于默认配置（相同牵引力，质量更大）
     站间运行时间比默认配置更长
```
