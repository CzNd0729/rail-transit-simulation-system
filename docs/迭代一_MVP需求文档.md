# 迭代一：MVP 最小可行系统 — 需求文档

> **目标：** 在最短的时间内跑通「单列车从 A 站到 B 站」的完整仿真闭环，包含**后端仿真引擎 + 前端可视化界面**，验证全链路可行性。
>
> **总原则：** 后端只实现两个核心子系统（轨道 + 车辆），供电与信号全部用静态数据替代。前端实现核心视图和交互控制，不追求完美 UI，只追求逻辑闭环和可视化验证。

## 二、后端模块详细需求

### 2.1 仿真主循环（Simulation Orchestrator）

| 需求编号 | 需求描述 | 验收标准 |
|---------|---------|---------|
| ORC-01 | 以固定时间步长推进仿真时钟（默认 100ms，可配置） | 每一步时钟推进等于设定步长 |
| ORC-02 | 每个仿真步依次调用：轨道查询 → 车辆计算 → 数据记录 | 调用顺序固定，无遗漏 |
| ORC-03 | 支持指定仿真总时长或仿真结束条件（如列车到站停车） | 仿真可在设定条件到达时自动停止 |
| ORC-04 | 每个步长结束时将关键状态写入日志/内存缓冲区 | 至少记录：时刻、列车位置、速度、加速度 |
| ORC-05 | 仿真结束后导出 CSV 格式的运行记录 | 可用 Excel 或 Python 直接读取绘图 |
| ORC-06 | 支持运行/暂停/停止/重置四种状态切换 | 状态机转换正确，无非法状态跳转 |
| ORC-07 | 支持仿真速度倍率 1× / 5× / 10×（通过 sleep 控制） | 切换倍率后仿真推进速度相应变化 |
| ORC-08 | 仿真过程中每步通过 WebSocket 推送状态快照到前端 | 前端能实时收到并渲染仿真数据 |

### 2.2 轨道系统（Track System）

> **设计原则：** 将真实的三维空间曲线简化为**一维公里标路径 + 分段参数表**。放弃毫米级精度，只保证区段级正确性。

#### 2.2.1 数据模型

```
Track:
  - stations: Station[]      # 车站列表（含站台中心里程）
  - segments: Segment[]      # 区间分段列表

Station:
  - id: string               # 车站 ID，如 "ST01"
  - name: string             # 车站名称，如 "北京站"
  - chainage: float          # 站台中心公里标（m）
  - dwell_time: float        # 站停时间（s）

Segment:
  - id: string               # 区段 ID
  - start_chainage: float    # 起点公里标（m）
  - end_chainage: float      # 终点公里标（m）
  - length: float            # 长度（m）= end - start
  - gradient: float          # 坡度（‰，上坡为正）
  - curvature: float         # 曲率半径（m，直线为 0 或无穷大）
  - speed_limit: float       # 最高限速（km/h）
  - tunnel: boolean          # 是否在隧道内
```

初始化时预设一条简单线路（不少于 3 站 2 区间），手动录入以上参数，不依赖外部文件读取。

#### 2.2.2 运行时查询接口

| 需求编号 | 需求描述 | 验收标准 |
|---------|---------|---------|
| TRK-01 | 给定列车当前公里标，返回列车所在区段 ID | 边界点优先归属起始区段 |
| TRK-02 | 给定列车当前公里标，返回该位置坡度（‰） | 精确到区段内任一位置一致 |
| TRK-03 | 给定列车当前公里标，返回该位置曲率半径（m） | 直线返回 `Infinity` |
| TRK-04 | 给定列车当前公里标，返回该位置限速（km/h） | 不得超过该区段限速 |
| TRK-05 | 给定公里标，判断是否为隧道区间 | 返回 boolean |
| TRK-06 | 给定车站 ID 和方向，返回站台中心公里标 | 支持上下行方向查询 |
| TRK-07 | 给定公里标，判断是否在站台范围内（±15m） | 用于触发停车逻辑 |
| TRK-08 | 返回全线车站和区间列表（供前端初始化渲染） | 前端可获取完整线路拓扑 |

**MVP 不实现：** 道岔任何逻辑、轨道电路/计轴器、三维曲线精确表达、竖曲线。

### 2.3 车辆系统（Vehicle System）

> **设计原则：** 实现最基本的单质点动力学模型。忽略轮轨粘着、空气制动、电气制动等复杂物理过程。

#### 2.3.1 数据模型

```
Vehicle:
  - id: string                          # 车辆 ID
  - mass: float                         # 当前总质量（kg, 含载客）
  - empty_mass: float                   # 空车质量（kg）
  - passenger_load: float               # 当前载客量（占满员比例 0.0~1.0）
  - max_traction_force: float           # 最大牵引力（N）
  - max_brake_force: float              # 最大制动力（N）
  - traction_curve: TractionCurve[]     # 牵引特性曲线（速度→牵引力百分比）

TractionCurvePoint:
  - speed: float                        # 速度（km/h）
  - force_percent: float                # 该速度下可用牵引力百分比（0~1.0）
```

#### 2.3.2 动力学解算

| 需求编号 | 需求描述 | 验收标准 |
|---------|---------|---------|
| VHC-01 | 根据当前速度查牵引特性曲线，计算实际牵引力（N） | 低速满牵引力，高速自然降功 |
| VHC-02 | 计算 Davis 基本阻力：\(R = A + Bv + Cv^2\)（A/B/C 可配） | 单位 N，速度单位 m/s |
| VHC-03 | 计算坡度附加阻力：\(R_g = m \cdot g \cdot \sin(\theta)\) | θ 从坡度‰换算 |
| VHC-04 | 计算弯道附加阻力：\(R_c = m \cdot g \cdot k / R\) | k 为经验系数（默认 600） |
| VHC-05 | 计算隧道附加空气阻力：若在隧道内，增加 1.2 倍空气阻力系数 | 仅在隧道区段生效 |
| VHC-06 | 合力计算：\(F_{net} = F_{traction} - F_{brake} - \sum R\) | 牵引为正，制动力负 |
| VHC-07 | 加速度计算：\(a = F_{net} / m\) | 单位 m/s² |
| VHC-08 | 速度积分（显式欧拉）：\(v_{t+1} = v_t + a \cdot \Delta t\) | 下限为 0，上限为限速 |
| VHC-09 | 位置积分：\(x_{t+1} = x_t + v_{t+1} \cdot \Delta t\) | 单位 m |
| VHC-10 | 到达终点站后速度置零，不再更新位置 | 仿真终止条件之一 |

#### 2.3.3 牵引特性曲线（MVP 简化版）

采用分段线性模型（默认参数可配置）：

| 速度区间 (km/h) | 可用牵引力百分比 |
|----------------|----------------|
| 0 ~ 40         | 100%            |
| 40 ~ 80        | 100% → 50% 线性递减 |
| > 80           | 50%             |

#### 2.3.4 基本阻力参数（Davis 公式默认值，可配置）

| 参数 | 默认值 | 说明 |
|------|-------|------|
| A    | 0.01 × m × g | 滚动阻力系数 |
| B    | 0.0001 × m × g | 线性项系数 |
| C    | 0.5 × ρ × Cd × A_front | 空气阻力（ρ=1.2, Cd=0.5, A_front=10m²） |

#### 2.3.5 输出记录

| 需求编号 | 需求描述 | 验收标准 |
|---------|---------|---------|
| VHC-11 | 每个仿真步记录：时间(s)、位置(m)、速度(km/h)、加速度(m/s²)、工况(牵引/惰行/制动) | 数据完整无跳步 |
| VHC-12 | 记录各阻力分量（基本阻力、坡度阻力、弯道阻力、隧道阻力） | 可溯源各阻力占比 |
| VHC-13 | 记录累计运行距离和总运行时间 | 用于计算平均速度 |

**MVP 不实现：** 轮轨粘着/防滑、电气制动→空气制动切换、多节车厢编组动力学、载荷转移、车门/屏蔽门逻辑。

### 2.4 静态数据提供器 — 供电

| 需求编号 | 需求描述 | 验收标准 |
|---------|---------|---------|
| PWR-01 | 受电弓端电压固定为 DC 1500V（可配置） | 列车计算牵引功率时始终使用该值 |
| PWR-02 | 忽略网压波动、再生制动回馈、变电所容量限制 | 不引入任何电压跌落逻辑 |

**MVP 不实现：** 潮流计算、再生制动能量吸收、变电所模型、网压波动。

### 2.5 静态数据提供器 — 信号

| 需求编号 | 需求描述 | 验收标准 |
|---------|---------|---------|
| SIG-01 | 列车运行模式为预设三段式：**牵引→惰行→制动** | 按里程触发模式切换 |
| SIG-02 | 牵引阶段：满牵引力加速至目标速度（默认 80% 限速） | 到达目标速度后自动切换 |
| SIG-03 | 惰行阶段：切断牵引力，依靠惯性滑行 | 仅在阻力作用下减速 |
| SIG-04 | 制动阶段：接近车站时施加最大制动力，使列车在站台中心 ±1m 内停车 | 超调范围 ≤ ±1m |
| SIG-05 | 制动触发点自动计算：根据当前速度和减速度能力反推 | 制动触发过早 → 惰行延长；过晚 → 增加制动力 |
| SIG-06 | 站停时间（dwell_time）内速度始终为 0 | 站停结束后自动进入下一区间 |

#### 2.5.1 三段式运行模式详细逻辑

```
每个区间循环执行：

  1. 牵引阶段
     条件：v < v_target (v_target = 0.8 × speed_limit)
     动作：施加最大牵引力
     结束：v >= v_target → 进入惰行

  2. 惰行阶段
     条件：已牵引至目标速度，且距离前方站台 > 制动触发距离
     动作：F_traction = 0, F_brake = 0
     结束：当前位置 + 制动触发距离 >= 站台中心位置 → 进入制动

  3. 制动阶段
     条件：当前位置 + 制动触发距离 >= 站台中心位置
     动作：施加最大制动力
     结束：v <= 0 且 |x - station_center| < 1m → 停车
```

**MVP 不实现：** ATP 安全包络计算、ATO PID/MPC 控制、ATS 运行图调整与晚点恢复、联锁逻辑、移动授权（MA）、紧急制动曲线。

### 2.6 数据记录与输出

| 需求编号 | 需求描述 | 验收标准 |
|---------|---------|---------|
| OUT-01 | 仿真结束后生成 CSV 文件，包含所有时间步的记录 | 文件可被 Excel/Python 直接打开 |
| OUT-02 | CSV 列：time(s), pos(m), speed(km/h), accel(m/s²), mode, grade_resist(N), curve_resist(N), tunnel_resist(N), basic_resist(N), total_resist(N) | 列定义完整 |
| OUT-03 | 输出关键统计值：总运行时间、总运行距离、平均速度、最大速度、最大加速度、最大减速度 | 统计值写入摘要文件或控制台 |

### 2.7 WebSocket 推送服务

| 需求编号 | 需求描述 | 验收标准 |
|---------|---------|---------|
| WS-01 | 仿真运行中每步将状态快照通过 WebSocket 推送到所有已连接客户端 | 推送频率与仿真步长一致 |
| WS-02 | 连接建立后立即发送初始化消息（含线路、车辆、仿真配置） | 前端可据此完成初始渲染 |
| WS-03 | 支持仿真控制指令的接收（运行/暂停/停止/重置/速度倍率） | 后端按指令切换仿真状态 |
| WS-04 | 支持参数更新指令的接收（车辆参数、线路参数、信号参数） | 参数更新后下一个仿真步生效 |
| WS-05 | 连接断开后自动清理，不影响仿真运行 | 无连接泄漏 |

## 四、验收场景（User Story）

### 场景 1：单列车一站直达（后端 + 前端联合验证）

```
给定：线路为 3 站 2 区间（A站→B站→C站），各区间参数已知
     列车空车质量 200t，AW2 载客（满员 60%）
     各区间限速 80km/h
     前端已连接后端 WebSocket
当：  用户点击"运行"按钮
则：  前端顶部栏时钟开始走动
     综合视图中：
       - 线路纵断面图显示三个车站和两个区间
       - 列车图标从 A 站出发沿线路移动
       - 速度-位置曲线实时绘制
       - 关键状态卡片更新数值
     车辆视图中：
       - 速度-时间曲线上升→平缓→下降
       - 加速度曲线显示正/负/零
       - 工况指示器：牵引→惰行→制动交替
     列车从 A 站出发
     → 牵引至 64km/h（80%限速）
     → 惰行一段距离
     → 制动减速至 B 站站台中心 ±1m 停车
     → 停车 30s（站停时间），时钟继续走动
     → 重复上述过程至 C 站
     → 仿真结束，状态栏显示"stopped"
```

### 场景 2：坡度影响验证（可视化对比）

```
给定：场景 1 的线路基础上，B站→C站区间加入 30‰ 上坡
当：  仿真启动
则：  车辆视图中速度-时间曲线在上坡段斜率明显变化
     阻力分解图中坡度阻力分量显著增加
     与场景 1 的 B站→C站段运行时间应有可量化的差异
```

### 场景 3：仿真控制交互验证

```
给定：仿真正在运行
当：  用户点击"暂停"按钮
则：  仿真时钟停止，列车位置/速度冻结
     所有曲线和数值停止更新
当：  用户点击"继续"按钮
则：  仿真从暂停处继续推进
当：  用户点击"停止"按钮
则：  仿真结束，显示运行统计摘要
当：  用户选择 10× 倍率后重新运行
则：  仿真推进速度明显加快（10 分钟运营 ≈ 1 分钟完成）
```

### 场景 4：参数编辑验证

```
给定：仿真处于空闲或暂停状态
当：  用户在参数面板中将车辆质量从 200t 改为 220t
     然后启动仿真
则：  加速度曲线明显低于默认配置（相同牵引力，质量更大）
     站间运行时间比默认配置更长
```

## 五、非功能需求

| 编号 | 需求描述 | 验收标准 |
|------|---------|---------|
| NFR-01 | 仿真时钟推进速度 ≥ 10× 实时（单列车） | 10 分钟的实际运营在 ≤ 1 分钟内算完 |
| NFR-02 | 后端代码语言为 Python 3.10+，仅依赖标准库 + FastAPI + uvicorn + pandas（仅用于 CSV 导出） | 最小化第三方依赖 |
| NFR-03 | 所有核心计算函数有单元测试覆盖 | 关键公式（阻力计算、速度积分）有 ≥ 80% 行覆盖率 |
| NFR-04 | 参数化设计：线路参数、车辆参数、仿真参数通过配置文件（YAML/JSON）或类构造参数传入 | 不改代码即可调整线路和车辆参数 |
| NFR-05 | 前端页面加载时间 ≤ 3s | Lighthouse 测试 |
| NFR-06 | WebSocket 推送延迟 ≤ 200ms（从引擎状态更新到前端渲染的时间差） | 在 1× 倍率下测试 |
| NFR-07 | 前端在有 WebSocket 推送时流畅运行 60fps | 无明显卡顿和掉帧 |
| NFR-08 | 实时曲线图支持数据点降采样（最多显示最近 2000 点） | 长时间仿真不卡顿 |
