# 司机台联动测试脚本

本目录包含独立的司机台联动测试脚本，用于测试仿真平台与司机驾驶模拟台之间的通信和联动控制。

## 协议概述

根据《轨交多系统平台接口协议汇总20260630.md》，司机台涉及以下通信协议：

| 协议 | 传输方式 | 服务器 | 端口 | 方向 |
|------|---------|--------|------|------|
| PLC协议 | TCP | 192.168.100.123 | 8001/8002/8003 | 上位机 ↔ PLC |
| 网络屏协议 | TCP | 192.168.100.122 | 8888 | 上位机 → 网络屏 |
| 信号屏协议 | TCP | 192.168.100.121 | 9999 | 上位机 → 信号屏 |
| 信号系统UDP | UDP | 动态 | 动态 | 信号系统 ↔ 总控数据库 |

## 文件结构

```
driver_cab_test/
├── __init__.py               # 包初始化
├── README.md                 # 本文件
├── config.py                 # 通信参数配置
├── protocols.py              # 协议编解码核心库（含自检）
├── plc_simulator.py          # PLC模拟服务器（模拟真实PLC）
├── cab_simulator.py          # 司机台整体模拟器（PLC+网络屏+信号屏）
├── test_signal_to_cab.py     # 测试：信号系统 → 司机台（数据下发，6场景）
├── test_cab_to_signal.py     # 测试：司机台 → 信号系统（控制上行，6场景）
└── test_full_loop.py         # 全链路联动测试（10阶段驾驶循环）
```

## 使用方式

### 1. 模拟模式（无真实硬件）

在同一台机器上运行模拟器 + 测试脚本：

```bash
# 终端1：启动司机台模拟器（模拟PLC+网络屏+信号屏）
python -m driver-cab-test.cab_simulator

# 终端2：运行全链路联动测试
python -m driver-cab-test.test_full_loop
```

### 2. 单独测试PLC

```bash
# 终端1：启动PLC模拟器
python -m driver-cab-test.plc_simulator

# 终端2：运行信号→司机台测试
python -m driver-cab-test.test_signal_to_cab
```

### 3. 连接真实硬件

修改 `config.py` 中的 `USE_REAL_HARDWARE = True`，并确保网络连通：

- PLC: 192.168.100.123:8001/8002/8003
- 网络屏: 192.168.100.122:8888
- 信号屏: 192.168.100.121:9999

## 测试场景

| 场景 | 说明 | 验证点 |
|------|------|--------|
| 场景1: 数据下发 | 信号系统模拟数据 → 司机台显示 | PLC接收46字节、网络屏接收572字节、信号屏接收66字节 |
| 场景2: 控制上行 | 司机台操作（手柄/钥匙/按钮）→ 信号系统 | 26字节控制报文解析、ATP安全/非安全输入位映射 |
| 场景3: 全链路联动 | 模拟完整驾驶循环：激活→牵引→巡航→制动→停站 | 各协议协同工作、数据一致性 |
| 场景4: 紧急制动 | 模拟紧急制动触发与复位 | ATP安全输出紧急制动位、PLC响应 |
| 场景5: ATO自动驾驶 | 模拟ATO模式下自动驾驶 | ATO输出命令、牵引/制动百分比联动 |
